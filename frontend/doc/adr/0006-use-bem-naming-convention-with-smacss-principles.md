# 6. Use BEM Naming Convention with SMACSS Principles

Date: 2026-03-22

## Status

Accepted

## Context

As applications grow, inconsistent naming and styling patterns lead to conflicts, duplication, and unpredictable behavior.

## Decision

We will use the BEM (Block, Element, Modifier) naming convention alongside SMACSS categorization (Base, Layout, Module, State, Theme).

## Consequences

* **Positive:** Predictable and scalable CSS structure.
* **Positive:** Reduced naming conflicts.
* **Positive:** Easier onboarding for new developers.
* **Negative:** Slight verbosity in class names.
