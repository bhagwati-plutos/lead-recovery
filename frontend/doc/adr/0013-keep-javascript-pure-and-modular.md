# 13. Keep JavaScript Pure and Modular

Date: 2026-03-22

## Status

Accepted

## Context

Large monolithic scripts are hard to test, debug, and maintain.

## Decision

We will write modular, reusable JavaScript functions and prefer pure functions where possible.

## Consequences

* **Positive:** Easier testing and debugging.
* **Positive:** Better reusability.
* **Positive:** Improved maintainability.
* **Negative:** Requires upfront design discipline.
